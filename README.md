## Featured project

### [relief-probe](https://github.com/owgreen-dev/relief-probe)

Fraud-lead research pipeline over **11.4M public PPP loans**, validated against real enforcement outcomes from DOJ/SBA-OIG sources.

This project focuses on turning public records into transparent, defensible investigative leads — not accusations.

**Highlights:**

- Built a local analytical warehouse from public SBA/DOJ data
- Used anomaly detection and ML experiments to rank potentially suspicious loans
- Added positive-unlabeled learning and LightGBM experiments
- Benchmarked signals against known prosecuted cases
- Included bootstrap confidence intervals and clear model limitations
- Built Streamlit views for analyst-style review
- Added LLM-assisted entity resolution, retrieval, and similar-case workflows
- Framed outputs responsibly as statistical leads, not proof of fraud

**Stack:** Python · DuckDB · pandas · scikit-learn · LightGBM · Streamlit · LLM workflows · graph/retrieval methods

## Organizations / workstreams

I use GitHub organizations to keep different product and research directions separated instead of mixing every experiment into one personal account.

### [Shipi18n](https://github.com/Shipi18n)

**Developer tooling for app localization and translation workflows.**

Shipi18n is focused on making internationalization faster for developers: translating locale files, preserving placeholders, supporting JSON/i18n workflows, and integrating translation into CLI, build, and CI/CD pipelines.

Projects in this org include:

- Node.js API client for the Shipi18n translation API
- CLI for translating locale files
- Vite plugin for build-time translation
- i18next backend integration
- GitHub Action for CI/CD translation workflows
- React, Vue, Next.js, and Node.js examples

This workstream is about making localization feel like part of the developer workflow instead of a manual copy/paste process.

### [Shipispec](https://github.com/Shipispec)

**Developer tooling for spec-driven and AI-assisted coding workflows.**

Shipispec is focused on tools that help developers recover, validate, and ship code generated by LLMs or spec-to-code systems.

The current flagship project is **tsfix**, a TypeScript repair tool for LLM-generated code. It combines deterministic TypeScript/VS Code-style quick fixes with optional LLM-based repair for harder cases.

Projects in this org include:

- TypeScript error recovery for LLM-generated code
- Library-aware repair hints for framework and dependency migrations
- CLI workflows for fixing `tsc --noEmit` failures
- CI-friendly validation and repair tooling
- Landing page work for spec-driven development tools

This workstream is about the gap between “the AI generated code” and “the code actually type-checks, respects installed libraries, and can move through CI.”
